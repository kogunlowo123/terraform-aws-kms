terraform-aws-kms

AWS KMS key management with multi-region replication, automatic rotation, aliases, grants, and CloudWatch alarms
Terraform ≥ 1.0 AWS Provider ≥ 5.0 Symmetric + Asymmetric Auto Rotation

Executive Summary

This module creates and manages AWS KMS keys supporting symmetric (ENCRYPT_DECRYPT), asymmetric (SIGN_VERIFY), and HMAC (GENERATE_VERIFY_MAC) key types. It includes automatic key rotation with configurable periods (90-2560 days), multi-region primary keys with replica deployment, multiple aliases, KMS grants for temporary programmatic access delegation, custom or default key policies, and CloudWatch metric alarms for key usage monitoring. The module follows the per-service key strategy recommended for compliance frameworks such as SOC 2, PCI-DSS, and HIPAA.

Usage Threshold

Component Breakdown

KMS Key

  • Symmetric (AES-256-GCM) or asymmetric (RSA/ECC)
  • HMAC key specs (224/256/384/512)
  • ENCRYPT_DECRYPT, SIGN_VERIFY, GENERATE_VERIFY_MAC usage
  • Configurable deletion window (7-30 days)
  • Automatic key rotation (90-2560 day period)

Multi-Region

  • Primary key with multi_region=true
  • Replica keys in specified regions via aws.replica provider
  • Same key material across regions
  • Independent key policies per region

Aliases & Grants

  • Multiple aliases per key (must start with alias/)
  • Grants for temporary access delegation
  • Encryption context constraints on grants
  • Retiring principal support for grant lifecycle

Monitoring

  • CloudWatch metric alarm for key usage
  • Threshold: 1000 operations per 5 minutes
  • Optional via enable_cloudwatch_alarms

Data Flow

  1. AWS services (S3, RDS, EBS, etc.) reference the KMS key ARN or alias for encryption
  2. KMS generates data encryption keys (DEKs) using the CMK for envelope encryption
  3. DEK encrypts the data; encrypted DEK is stored alongside
  4. For decryption, KMS decrypts the DEK using the CMK, then DEK decrypts the data
  5. Grants allow specific IAM principals to use the key for defined operations
  6. Multi-region replicas enable cross-region encrypt/decrypt without key transfer
  7. Key rotation automatically creates new key material while preserving old material for decryption

Industry Adaptation

Per-Service Key Strategy

  • alias/prod-s3-data-at-rest -- S3 encryption
  • alias/prod-rds-encryption -- RDS instances
  • alias/prod-lambda-env-vars -- Lambda env vars
  • alias/prod-secrets-manager -- Secrets Manager
  • Separate keys reduce blast radius

Compliance (SOC 2, PCI-DSS, HIPAA)

  • Customer-managed keys with custom policies
  • Independent rotation per workload
  • CloudTrail audit per key for granular logging
  • Grants for temporary cross-account access

Multi-Region / DR

  • Primary + replica for cross-region encryption
  • Same key ID enables seamless failover
  • No key material transfer required

Production Readiness Checklist

  • Enable enable_key_rotation = true
  • Set deletion_window_in_days = 30 (maximum)
  • Define a custom key policy (avoid default root access in production)
  • Use meaningful aliases following naming convention
  • Create separate keys per service / workload
  • Enable CloudWatch alarms for usage monitoring
  • For multi-region: configure replica in DR region
  • Scope grants with encryption context constraints
  • Tag all keys with Environment, Service, Owner
